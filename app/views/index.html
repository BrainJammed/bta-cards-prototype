1 - Cards design requirements

Design Requirements for the Cards

When designing the cards please follow these requirements

all card elements should be the same height

Each card should include:

card-body *required

  • use all available space
  • card title (Options: hyperlink, no hyperlink) *required
  • status (Options: status-low, status-med, status-high)
    • if status is included it should be displayed directly below the title
  • detail (Options: detail, detail-large)
  • description

card-actions

if included should be the same height

  • hyperlink (Maximum 5)

Page layout:

  • display all cards at the same height
  • should fit within a responsive grid layout
    • each card should be a maximum of 1/3 page width

The cards should all:

  • have a card-body
  • if card-body includes a hyperlink
    • have hover state
    • when clicked redirects the user to another page
  • work with JS turned off

The cards should be fully accessible:

  • using a screenreader
  • using keyboard only navigation
  • screen magnifiers

The cards should work on mobile devices:

  • they should stack to a single govuk-column
